# Phase 1: 仕様書（spec.md）の作成

## 入力

`dev/prompt.md` の内容を読み込む：

@dev/prompt.md

## タスク

上記の `dev/prompt.md` を分析し、`dev/spec.md` を対話的に作成する。

### spec.md の構成テンプレート

```markdown
# {プロジェクト名} - 仕様書

## 概要
（分析対象・目的を1〜2文で説明）

## 成果物
- `src/analysis.ipynb` - 分析ノートブック（上から順に再現可能）
- `article/output.md` - 記事（常体）

## 技術スタック
（使用ライブラリ・ツールの一覧）

---

## ステップ

### Step 0: 環境準備
- [ ] ...

### Step 1: データ取得・前処理
- [ ] ...

（中間ステップ）

### Step N: 記事作成
- [ ] `article/output.md` に分析結果を記事としてまとめる

---

## データ仕様
（テーブル定義・カラムの説明）

## 注意事項
（分析上の制約・方針）
```

### 作成ルール

- ステップは `[ ]` チェックリスト形式で記述する
- 各 Step は1つの目的に絞り、実装判断が必要な箇所は具体的に記述する
- `prompt.md` の記事骨子・手法セクションの内容をステップに落とし込む
- 「データの絞り込み条件」「ブランド集約方針」などの分析上の注意点は末尾の注意事項にまとめる

## 手順

1. `dev/prompt.md` から分析計画・記事骨子・データ情報を読み取る
2. `dev/spec.md` のドラフトを作成してユーザーに提示する
3. ユーザーのフィードバックを反映して修正する
4. 確定したら `dev/spec.md` に保存する

完了後: `/analyze-code` で次フェーズへ
