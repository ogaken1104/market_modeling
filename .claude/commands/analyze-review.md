# Phase 4: 記事レビュー

## 入力

レビュー対象の記事を読み込む：

@article/output.md

---

分析の仕様・背景を確認する：

@dev/spec.md

## タスク

`article/output.md` をプロのデータサイエンティスト視点でレビューし、修正点を列挙する。

### レビューの観点

以下の観点で問題点・改善点を洗い出す。

1. **統計的正確性**: 数式・定義・パラメータ解釈に誤りや不正確な表現がないか
2. **論理の一貫性**: 仮定→導出→解釈の流れに飛躍や矛盾がないか
3. **数値の整合性**: 本文中の数値が分析結果と一致しているか
4. **モデルの限界の記述**: 適切に限界・前提条件を説明しているか
5. **文章の明確さ**: 専門用語の定義、曖昧な表現、冗長な記述がないか
6. **参考文献**: 本文中の主張と参照元が対応しているか

### レビューの作法

- **フラットに列挙する**。過度な肯定・否定は不要
- 各指摘は「箇所の特定 → 問題点 → 改善案」の形式で記述する
- 重大度を `[重要]` / `[軽微]` で示す
- 修正を加える前に、必ずユーザーに確認する

## 手順

1. `article/output.md` を通読し、上記観点で問題点を洗い出す
2. 修正点のリストをユーザーに提示する
3. ユーザーが修正を要請した項目のみ `article/output.md` を編集する
4. 編集後、該当箇所をユーザーに確認する
